这节讲的是具体的设备的配置操作方法
设备配置方式：console auxiliary telnet tftp web

模式：
用户模式
特权模式
全局模式
接口模式

? 帮助
默认是用户模式
enable 进入特权模式
show running-config 查看正在运行的配置
config t 进入全局配置模式
interface 进入接口
do show ip int bri 查看接口
ip address 192.168.1.1 255.255.255.0 配置IP地址
exit 退出
end 直接退出到特权模式
默认接口是关闭的
no shutdown 启用接口
no hostname 删除主机名
no ip address 删除IP地址
no 删除命令（后面跟具体要删除的内容）

ctrl+ shift + 6 输入错误命令时取消域名解析过程（域名解析超时的时间较长）
no ip domain-lookup 关闭域名解析（这样在输入错误命令不会进行域名解析）
show terminal 查看控制台
terminal history size 100 设置历史大小
banner login # xuhongbin # 设置登陆提示信息
banner motd 设置每日信息
description string 给接口设置描述信息
write 保存
copy running-config startup-config 保存配置到开机启动配置
show startup-config 查看开机启动配置

路由器密码重启过程：
1 重启路由器，按ctrl+break c/b,进入remmon模式
2 confreg 0x2142　修改寄存器的值
3 reset　重启
4 copy start run　把nvram内的文件转到内存中
5 config-register 0x2102
6 修改密码
7 copy run start
8 reload NVRAM：config.text
0x2142:路由器启动不读取NVRAM，出厂配置启动
0x2102:路由器启动读取NVRAM，正常配置启动
0x2101:路由器启动不读取新指定的IOS，以出厂IOS启动，通常
为base版本，支持功能有限

交换机密码恢复过程：
1 重启，按面板上的mode键3-5秒
2 进入switch：模式
3 输入：flash_init 初始化flash文件系统
4 dir flash:查看flash内文件，其中包含配置文件为config.text
5 rename flash:config.text flash:config.old
把原来的文件重命名，随意起名字
6 boot重启
7 rename flash:config.old flash:config.text
8 copy flash:config.text system:running-config
9 修改密码
10 copy run start 保存配置
11 reload/reset/boot 重启
