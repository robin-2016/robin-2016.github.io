write-once-publish-everywhere

AI写作自动化发布

一句话写稿配图，自动发全网平台

摘要

多平台分发内容的人都遇到过同一个麻烦：一篇稿子写完，还要手动改格式、传封面、逐个平台粘贴发布，一圈下来比写稿本身还耗时间。write-blog 是运行在 Claude Code 里的一个写作发布 skill，一句话或一个链接就能生成成稿、配好封面图和知识总结图，再自动把草稿铺到公众号、知乎、掘金、B站等十几个平台，人只需要在关键节点点头确认。

核心优势

三种输入方式，覆盖不同场景。 daily-news 模式自动抓取当天 Hacker News、GitHub Trending、Reddit、X 的热点全量列表，交给用户手动挑主题，避免自动选到不感兴趣的话题；github 模式丢一个仓库链接进去，自动抓取项目信息重新组织成原创中文文章；sentence 模式最灵活，一句自然语言描述就能触发写作，必要时自动联网搜索补充材料。

自带事实核查环节。 sentence 模式因为没有单一权威信源，风险最高，所以多了一道工序：逐条核对文中的版本号、数据、对比结论，标成”确认无误 / 已过时 / 无法证实 / 明显错误”四类，有问题直接改好，不用用户回头再挑错。

双配图自动生成，不用等设计。 封面图调用 HuggingFace 的 FLUX.1-schnell 模型生成 16:9 电影感画面；知识总结图调用 DashScope 的图像模型，把文中 3-5 个要点提炼成一张 3:4 竖版信息图，还会根据内容类型自动切换极简科技风、杂志排版风、手绘信息图风三种视觉风格——技术教程和财经资讯配出来的图长得不一样。

一稿真的能发十几个平台。 微信公众号走官方 API 建草稿；知乎、掘金、CSDN、B站、百家号、cnblogs、OSChina、微博、头条、抖音、segmentfault 这些平台，靠一个开源的 Chrome 扩展直接复用用户自己浏览器里的登录态调用官方接口，不经过任何第三方服务器，再用自动化脚本逐个点发布。财经、时评这类内容还能单独走一条只发知乎/抖音/头条/微博/百家号的精简路径，不用挤在技术社区里显得突兀。

面向人群

  • 个人多平台内容创作者、自媒体作者：一篇稿子想同时发知乎专栏、掘金、公众号、微博，重复的格式转换和逐平台粘贴操作可以完全省掉。
  • 企业和产品团队的内容运营：新功能上线、版本发布这类内容想同时铺技术社区和大众平台做曝光，技术向和大众向内容还能分别配置不同的发布平台组合，一次投入覆盖更广的受众。

快速上手

在 Claude Code 里直接调用对应模式即可：

# 从今日热点里选题
/write-blog daily-news

# 从一个 GitHub 项目生成文章
/write-blog github https://github.com/owner/repo

# 用一句话描述主题
/write-blog sentence 写一篇介绍 xxx 工具的文章

生成流程会依次跑完成稿、事实核查（仅 sentence 模式）、封面图、知识总结图，然后停下来给用户看效果——文章、封面图、知识图一起展示，确认没问题或者提出修改意见，改完再继续。

进阶用法

发布环节明确分成两条路径，说”发布”默认走全平台路径，只有用户明确说”发布到非技术平台”才会切换到财经/时评专用的精简平台组合，不会靠主题自动猜测——这个判断权始终留给用户。小红书由于自动化导入链接的方式曾被平台识别为异常行为、出现过封号预警，目前保持人工发布，用户自己手动从公众号平台导入。整个发布链路依赖的都是用户自己已登录的浏览器会话和自己申请的 API key（HuggingFace 生成封面、DashScope 生成知识图、公众号自己的 AppID/AppSecret），不会代存代传任何账号密码，目前只在 Claude Code 环境里做过完整验证。
