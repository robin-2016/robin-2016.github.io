Claude Code开发工具

这套开源 Skills 让 AI 写代码像资深工程师

摘要

知名 TypeScript 讲师 Matt Pocock 把自己每天在用的一整套 Claude Code Skills 开源了。这不是又一个”一键生成全部代码”的框架,而是一组小巧、可组合、能拆开单独用的工程实践套件,专门解决用 AI 写代码时最容易踩的四个坑:需求对不齐、上下文啰嗦、代码跑不起来、项目越改越乱。目前已有约 6 万名开发者通过他的 newsletter 持续跟进这些 skills 的更新。

核心优势

  • 先”审问”再动手:/grill-me 和 /grill-with-docs 会在写代码前对你连环追问,把模糊需求逼成明确决策,从根源上减少”AI 理解错了”这种返工。
  • 共享语言,减少啰嗦:通过 CONTEXT.md 和 ADR(架构决策记录)沉淀项目黑话,比如把”某个课程里的某节课在文件系统里被具体化”简化成”物化级联”(materialization cascade)。命名统一了,AI 读代码更快,想得也更少,自然更省 token。
  • 红绿重构闭环:/tdd 强制先写失败测试再修复,diagnosing-bugs 把”复现 → 最小化 → 假设 → 插桩 → 修复 → 回归测试”这套调试纪律固化成流程,给 AI 稳定、可验证的反馈,而不是让它”盲写”。
  • 天天治理架构,不欠债:/improve-codebase-architecture 定期扫描代码库,找出可以”加深模块”的地方——用更小的接口封装更多行为,防止项目在 AI 加速开发的同时加速腐化成一坨”史莱姆代码”(ball of mud)。

面向人群

  • 已经在用 Claude Code / Codex 等编码 Agent,但常常觉得”它写的不是我想要的”的开发者
  • 想把项目里的领域黑话、架构约定沉淀成可复用文档,而不是每次都让 AI 从头猜的团队
  • 关心代码质量、想要 TDD 和持续架构治理,而不只是”能跑就行”的工程师
  • 想按需拆开使用单个 skill、而不是整套接入某个重量级框架的实用主义者

快速上手

一条命令即可安装:

npx skills@latest add mattpocock/skills

安装器会让你挑选想要的 skills,以及要装到哪些编码 Agent 上——记得勾选 /setup-matt-pocock-skills。装完之后在你的 Agent 里跑一次:

/setup-matt-pocock-skills

它会依次询问你用哪个 issue 追踪工具(GitHub、Linear 或本地文件)、triage 时用什么标签、文档存在哪里,配置完就可以直接用了。

进阶用法

Skills 分两类,按”谁能调用”区分:

  • User-invoked(需要你主动输入才触发,负责编排流程):比如 /grill-me、/to-prd、/to-issues、/triage。
  • Model-invoked(你可以手动调,Agent 也会在任务匹配时自动调用,承载可复用的具体纪律):比如 tdd、diagnosing-bugs、domain-modeling、code-review。

推荐的组合打法:先用 /grill-with-docs 把需求和领域模型一起理清楚,再交给 /tdd 按红绿重构一块块实现,遇到疑难 bug 用 diagnosing-bugs 走完整的诊断闭环,每隔几天跑一次 /improve-codebase-architecture 给代码库”体检”。四个环节分别对应前面提到的四个典型失败模式,组合起来就是一套从”对齐需求”到”长期维护”的完整闭环。
