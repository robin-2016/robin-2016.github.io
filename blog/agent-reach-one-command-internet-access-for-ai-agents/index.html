Agent Reach：一键给 AI 装上互联网能力

摘要

Agent Reach 是一个开源 CLI 工具，定位是“能力层”而非又一个爬虫或 API 封装——它负责替 AI Agent 选型、安装、体检、路由，把读推特、搜 Reddit、看 YouTube 字幕、刷小红书、查雪球行情这些原本要各自踩坑配置的事情，统一成一句话交给 Agent 自己搞定。项目全部依赖开源工具和免费 API，唯一可能的花费是服务器代理（约 $1/月），本地电脑完全免费。

是什么

Agent Reach 不直接实现“读取”逻辑，而是维护一份“每个平台该用哪个上游工具”的路由表：

channels/
├── web.py          → Jina Reader
├── twitter.py      → twitter-cli ▸ OpenCLI ▸ bird
├── youtube.py      → yt-dlp
├── github.py       → gh CLI
├── bilibili.py     → bili-cli ▸ OpenCLI ▸ 搜索 API
├── reddit.py       → OpenCLI ▸ rdt-cli
├── xiaohongshu.py  → OpenCLI ▸ xiaohongshu-mcp ▸ xhs-cli
└── ...

每个渠道都是“首选 + 备选”的有序列表，agent-reach doctor 会真实探测每个候选后端是否可用，而不是只看命令存不存在。某个接入方式被平台封了（作者举了个例子：2026 年 6 月 yt-dlp 被 B 站风控 412 封死），只需要把路由表里的顺序换成 bili-cli，用户完全无感——这也是它区别于“装一个爬虫工具”的核心思路：工具会过时，路由不会。

核心优势

  • 一句话安装：把安装文档链接丢给 Agent，Agent 自己读文档、装依赖、探测环境、注册 SKILL.md，之后遇到“全网调研”“搜推特”“看视频”这类需求会自动知道调哪个工具
  • 零配置起步：网页阅读、YouTube 字幕、RSS、GitHub 公开仓库、B站搜索、V2EX、雪球行情等渠道装完即用，不需要任何登录
  • 需要登录的按需开启：Twitter、小红书、Reddit、Facebook、Instagram、LinkedIn 这些平台需要 Cookie/登录态，Agent 会列出菜单，用户点名才装，不会一次性把所有权限都要走
  • 自带诊断和预览：agent-reach doctor 一条命令看清每个渠道当前走的是哪条后端；agent-reach install --dry-run 可以先预览会做哪些操作，--safe 模式则完全不自动改系统
  • 凭据只存本地：Cookie/Token 存在 ~/.agent-reach/config.yaml，文件权限 600（仅所有者可读写），不上传不外传，代码完全开源可审查

面向人群

  • 想让 Claude Code / OpenClaw / Cursor / Windsurf 等 Agent 具备“上网调研”能力、但不想一个个平台踩坑配置的开发者
  • 需要 Agent 定期抓取多平台舆情（产品评价、竞品动态、技术讨论）的运营和研究人员
  • 维护自己 Agent 工具链、希望“平台接入方式过时了不用自己重写”的技术团队

需要特别注意的一点：项目文档自己也明确提醒——Twitter、小红书等需要 Cookie 登录的平台，脚本化调用存在被平台风控检测、限制甚至封号的风险，官方建议务必使用专用小号，不要用主账号，原因是一旦 Cookie 泄露，小号能限制影响范围。这一点和我们之前处理小红书自动发布被限流预警时的经验是一致的，值得认真对待。

如何使用

一句话安装（复制给你的 AI Agent）：

帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md

Agent 会自动完成：安装 CLI（pip install agent-reach，自带 yt-dlp、feedparser）→ 检测并安装 Node.js/gh CLI/mcporter 等系统基建 → 通过 MCP 免费接入 Exa 搜索 → 判断本地/服务器环境给出配置建议 → 注册 SKILL.md → 询问是否需要登录类渠道。

担心安全，想先看看会做什么：

帮我安装 Agent Reach（安全模式）：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md
安装时使用 --safe 参数

安全模式不会自动修改系统，只会列出需要什么，由你自己决定装不装。也可以用 agent-reach install --dry-run 只预览不改动。

装好之后，不需要记具体命令，直接跟 Agent 说需求即可，例如：

  • “帮我看看这个 YouTube 视频讲了什么”
  • “B站搜一下 AI 教程”
  • “全网搜一下最新的 LLM 框架对比”

需要登录态的平台，告诉 Agent「帮我配 Twitter」/「帮我配小红书」，它会一步步引导你完成 Cookie 或浏览器登录态配置——如果是这类平台，建议先准备一个专用小号再配置，不要直接用主账号。

检查状态：

agent-reach doctor

卸载（会清除所有 token/cookie 和已注册的 skill 文件）：

agent-reach uninstall
